# AI Insights V2: Hybrid Architecture (Rule Engine + Cache + LLM)

## 🎯 Цель
Устранить прямую зависимость от Groq AI, снизить задержки (latency), сократить расходы на токены и повысить надежность системы (SLA).

## 🏗 Архитектура
Внедрен `LlmAdapter` — единый шлюз для всех AI-запросов, работающий по паттерну:
1. **Rule Engine (Rust)**: Мгновенная классификация известных ингредиентов (0 мс, 0 токенов).
2. **AI Cache (PostgreSQL)**: Кэширование успешных ответов LLM с TTL (1-2 мс, 0 токенов).
3. **Groq AI (LLM)**: Обработка только сложных/неизвестных запросов со строгим SLA (таймауты 3-10 сек).

## 🛠 Реализованные компоненты

### 1. Rule Engine (`classification_rules.rs`)
- Жестко закодированные правила на Rust для базовых категорий (Dairy, Meat, Vegetables, Fruits, Seafood, Grains, Beverages).
- Позволяет обходить LLM для 80% стандартных продуктов.

### 2. AI Cache (`ai_cache_repository.rs`)
- Таблица `ai_cache` в PostgreSQL.
- Хранит хэш промпта и ответ.
- Поддерживает TTL (Time-To-Live): 30 дней для инсайтов, 90 дней для классификации.

### 3. SLA & Timeouts (`llm_adapter.rs`)
- Использование `tokio::time::timeout` для предотвращения зависаний.
- Разные таймауты для разных задач:
  - Классификация: 3 секунды.
  - Перевод: 5 секунд.
  - Инсайты рецептов: 10 секунд.

### 4. AI Usage Metrics (`ai_usage_stats_repository.rs`)
- Таблица `ai_usage_stats` для мониторинга.
- Логирует: тип задачи, потраченные токены (prompt/completion), время выполнения (duration_ms).

### 5. Confidence Layer
- LLM теперь возвращает поле `confidence` (0.0 - 1.0).
- Если `confidence < 0.8`, система логирует предупреждение (потенциально требует ручной проверки).
- Бизнес-логика (например, расчет `feasibility_score`) перенесена из LLM в детерминированный код на Rust.
